1. 自动化消息与数据监控 (CRM 引擎)
实时消息抓取：通过 UI 自动化持续抓取微信聊天窗口的最新消息和发送者昵称。

活跃度统计：默默记录每个群成员的发言次数，并在内存中进行按日归档。

花名册自动更新：每天早上 08:00 会自动触发 UI 滚动，抓取并更新当前群聊的所有成员名单。

潜水人员排查：每天定时生成《群活跃度报表》，统计出今日活跃榜，并对比花名册找出“近7天潜水人员（零发言）”。

2. AI 智能问答系统
上下文感知回复：接入了 DeepSeek 大模型的 API。当有人在群里发送包含特定暗号（默认是 "@bot"）的消息时，程序会将最近的群聊记录作为上下文一并发送给 AI，让 AI 能“连贯且自然”地回答问题。

并发控制：设置了最大 AI 线程数（MAX_AI_THREADS = 2），防止群内多人同时触发导致系统卡死或 API 频率超限。如果排队满了，会给出友好提示。

3. 内置特定指令响应
除了 AI 回复，程序内置了几个快速触发的本地指令：

规则：回复群规信息（如入群/发言要求）。

颜值担当：一个活跃气氛的小功能，会自动计算近期发言最频繁的用户，并将其指定为“颜值担当”。

4. 强大的稳定性与容灾机制 (终极免维护版特性)
数据安全与备份：使用原子写入（先写临时文件 .tmp 再替换主文件）以及双重备份（.bak）机制，防止断电或意外导致 JSON 数据库损坏。

内存防爆（瘦身机制）：每天自动清理 35 天以前的历史统计数据，防止程序长期运行导致内存和文件过大。

UI 操作线程锁：由于操作微信窗口（复制粘贴、点击发送）是独占式动作，程序使用了 threading.Lock()，确保同一时间只有一个线程在控制微信输入框，避免消息乱码或串号。

异步缓冲落盘：消息不会每收到一条就读写一次硬盘，而是满 100 条或满 5 分钟统一保存一次，极大降低了 IO 消耗。
